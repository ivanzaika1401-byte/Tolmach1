# Толмач — живой переводчик переговоров (китайский ↔ русский)

Android-приложение для деловых встреч с китайскими партнёрами. Телефон лежит на столе, вы — в Bluetooth-наушниках.

**Как это работает на встрече:**

1. Нажимаете зелёную кнопку-сферу «Слушать китайский». Партнёр говорит — телефон распознаёт речь, переводит и тихо озвучивает русский перевод **вам в наушники**. Собеседник ничего не слышит.
2. Хотите ответить — кнопка «Ответить» слева: говорите по-русски, телефон озвучивает фразу **по-китайски через динамик** для партнёра, затем сам возвращается к прослушиванию.
3. Вся беседа — лентой: перевод крупно, оригинал под ним, время. Нажатие на реплику копирует её, кнопка-динамик в реплике озвучивает её ещё раз. В меню «Ещё» — стенограмма переговоров (в мессенджер или на почту) и очистка.

**Возможности:**

- Перевод работает **офлайн** (модели ML Kit скачиваются один раз) — критично для КНР, где сервисы Google недоступны.
- **Выбор языка собеседника** (шестерёнка): путунхуа / тайваньский мандарин / кантонский. Выбор виден в приборной строке и сохраняется.
- **Словарь терминов** — свои замены в переводе: марки, артикулы, Incoterms.
- **Повторная озвучка любой реплики** — партнёр переспросил? Нажмите динамик на своей фразе, телефон повторит её по-китайски.
- Полудуплекс против эха; собеседнику дают договорить (пауза до ~1,2 с не рвёт фразу); китайский синтез чуть замедлен для разборчивости.
- Экран не гаснет во время сеанса; живая строка показывает распознавание до конца фразы.
- Кнопка повторной загрузки моделей, если первый раз не вышло.

**Дизайн:** единая тёмная тема (системные полосы Android перекрашены в цвет приложения — никаких белых полос), фирменный шрифт Golos Text с полной кириллицей, один акцентный цвет — нефрит, пульт управления с пульсирующей кнопкой-сферой по центру, приборная строка статусов из трёх ячеек. Шрифт — по лицензии SIL OFL (файл LICENSE-GolosText-OFL.txt).

## Диалекты и качество перевода — честно

- «Все диалекты» не распознаёт ни одна система в мире. Деловые переговоры в Китае ведутся на путунхуа — его приложение понимает отлично (движок тот же, что у Google). Кантонский — выбирается в настройках, перевод с него слабее. Шанхайский, сычуаньский и прочие региональные диалекты не поддерживает никто; попросить партнёра перейти на путунхуа — обычная вежливая просьба.
- Офлайн-перевод zh↔ru в ML Kit идёт через английский: смысл надёжно, формулировки проще. Облачные движки точнее, но платные и в КНР без VPN не работают.
- **Юридически значимые условия (цены, сроки, штрафы) перепроверяйте с живым переводчиком.**

## Как получить установочный APK (≈10 минут, ничего не устанавливая)

В проект вшит облачный сборщик GitHub Actions — APK компилируется на серверах GitHub бесплатно.

1. Зарегистрируйтесь на **github.com**.
2. **New repository** → имя `tolmach` → *Create repository*.
3. *Uploading an existing file* → перетащите **всё содержимое распакованного архива** → *Commit changes*. Папку `.github` браузер может не перенести — тогда создайте файл вручную: *Add file → Create new file*, имя `.github/workflows/build-apk.yml`, вставьте содержимое одноимённого файла из архива.
4. Вкладка **Actions** → при первом заходе разрешите workflows → сборка «Build APK» запустится сама (или *Run workflow*). 5–8 минут до зелёной галочки.
5. Откройте завершившийся запуск → блок **Artifacts** → скачайте `Tolmach-APK` (zip, внутри `Tolmach.apk`).
6. Перекиньте APK на телефон, откройте, разрешите установку из неизвестных источников.

### Уже загружали проект раньше? Как обновить

1. В своём репозитории: **Add file → Upload files**.
2. Перетащите всё содержимое нового архива **кроме папки `.github`** (сборщик у вас уже есть, менять его не нужно — он подходит и для этой версии).
3. *Commit changes* → Actions соберёт новый APK сам → скачайте артефакт → установите APK поверх старого, данные и словарь сохранятся.

Сборка упала с красным крестом? Откройте её, скопируйте текст ошибки и пришлите в чат — исправлю.

### Альтернатива: локальная сборка

Android Studio → *Open* папку проекта → дождаться синхронизации → *Build → Build APK(s)*.

## Первый запуск — дома, с интернетом

1. Разрешите доступ к микрофону.
2. Дождитесь «Перевод: готов» в приборной строке (модели ~60–80 МБ, лучше Wi-Fi). Не вышло — «Повторить загрузку моделей».
3. Проверьте китайский голос: «Ответить» → скажите «Здравствуйте, рад знакомству» — телефон должен произнести это по-китайски. Молчит — Android → *Специальные возможности → Синтез речи* → докачайте китайский.
4. **Перед Китаем:** скачайте заранее офлайн-пакеты распознавания (настройки Google → Голосовой ввод → Распознавание речи офлайн → китайский и русский) — там Google заблокирован.

## Советы для переговоров

- Телефон микрофоном к собеседнику, экраном к себе.
- Просите говорить фразами по 1–2 предложения — перевод точнее и быстрее.
- Наушники перехватили микрофон и телефон плохо слышит партнёра? Отключите наушникам профиль «Телефон/звонки» в Bluetooth, оставьте «Медиа».
- Термины сделки — заранее в «Словарь»; стенограмму — себе на почту сразу после встречи.

## Ограничения

- Распознавание — системный сервис Google: онлайн отлично, офлайн-качество зависит от телефона.
- Маршрутизация китайского строго в динамик — Android 9+; на Android 8 стандартный маршрут.
- Цикл «фраза → перевод в ухе» ≈ 1,5–3 секунды — это норма.

## Техническая начинка

Kotlin 2.0, Jetpack Compose (Material 3), MVVM, шрифт Golos Text (статические начертания 400/500/600/700, SIL OFL). Распознавание — `SpeechRecognizer` (zh-CN / zh-TW / yue-Hant-HK / ru-RU, частичные результаты, удлинённая пауза конца фразы). Перевод — ML Kit Translation офлайн. Озвучка — `TextToSpeech`: китайский синтезируется в файл и играется через `MediaPlayer` с принудительной маршрутизацией на динамик (`setPreferredDevice`), русский — стандартным маршрутом в наушники. minSdk 26, targetSdk 34, версия 1.2.0.
