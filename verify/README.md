# Полнопроектная верификация без Android SDK

`run.sh` — две ступени на любом kotlinc:

1. **Типы всего проекта.** Все исходники — четыре движка, мозг
   (TranslatorViewModel), разговорник, тема и весь Compose-интерфейс
   (четыре вкладки, ~2450 строк UI) — компилируются против рукописных
   заглушек в `stubs/` (44 файла: Android, Compose runtime/ui/foundation/
   material3/animation, WebRTC, ML Kit, корутины). Ноль ошибок = каждый
   вызов, лямбда, перегрузка и suspend корректны.
2. **Поведение.** Мозг запускается по-настоящему: 47 живых проверок: трамплин
   корутин (`startCoroutine`) исполняет suspend-ветки перевода по-настоящему,
   фейк-переводчик в стабе ML Kit детерминирован, мини-JSON в `org_json`
   полнофункционален (сериализация + парсер) — лента и свои фразы проходят
   честный круг «сохранить → пересоздать ViewModel → прочитать дословно»;
   WebRTC-стаб симулирует сигнализацию (SDP, ICE COMPLETE, CONNECTED) —
   рукопожатие звонка A↔B исполняется целиком, с реальным кодированием
   и декодированием кодов обеими сторонами.

Копии исходников готовит `strip_commas.py`: убирает завершающие запятые
(синтаксис Kotlin 1.4+) и токен `@Composable` (для типовой проверки
аннотация инертна). Стабы намеренно минимальны: новый вызов внешнего API
уронит ступень 1, и заглушка дополняется осознанно.
Пройдено на Толмаче 4.0.2: 0 ошибок, 47/47.
